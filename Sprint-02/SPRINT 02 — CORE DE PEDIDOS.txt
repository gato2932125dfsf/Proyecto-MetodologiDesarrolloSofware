================================================================================
                    SPRINT 02 — CORE DE PEDIDOS
================================================================================
> Desarrollo del módulo transaccional del portal del cliente: registro y
  programación de pedidos de recarga de gas.

--------------------------------------------------------------------------------
[1] INFORMACIÓN GENERAL
--------------------------------------------------------------------------------
  * Sprint       : 02 de 06
  * Nombre       : Core de pedidos
  * Duración     : 2 semanas
  * Metodología  : Scrum
  * Estado       : Planificado

--------------------------------------------------------------------------------
[2] OBJETIVO DEL SPRINT
--------------------------------------------------------------------------------
  Construir el módulo transaccional que permite al cliente registrar pedidos
  de recarga, seleccionar el tipo de balón y programar la fecha de despacho,
  dejando la base para la asignación y el seguimiento en sprints posteriores.

--------------------------------------------------------------------------------
[3] REQUISITOS FUNCIONALES CUBIERTOS
--------------------------------------------------------------------------------
  * RF-09  Registro de pedido (cliente, cantidad, tipo de balón 10/15/45 kg,
           dirección, modalidad de pago)
  * RF-13  Notificación al cliente sobre el estado de su pedido
  * RF-06  Actualización y eliminación de datos de clientes
  * RF-07  Historial de pedidos por cliente

--------------------------------------------------------------------------------
[4] ACTIVIDADES PRINCIPALES
--------------------------------------------------------------------------------
  [+] Desarrollo del módulo transaccional del portal del cliente.
  [+] Registro de pedidos: cliente, cantidad y datos de entrega.
  [+] Selección del tipo de recarga (10 kg, 15 kg, 45 kg).
  [+] Programación de fechas de despacho.
  [+] Integración del frontend con la API de Supabase (lectura/escritura
      de pedidos).
